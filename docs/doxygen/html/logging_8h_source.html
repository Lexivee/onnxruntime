20 /*
21 
22  Logging overview and expected usage:
23 
24  At program startup:
25  * Create one or more ISink instances. If multiple, combine using composite_sink.
26  * Create a LoggingManager instance with the sink/s with is_default_instance set to true
27  * Only one instance should be created in this way, and it should remain valid for
28  until the program no longer needs to produce log output.
29 
30  You can either use the static default Logger which LoggingManager will create when constructed
31  via LoggingManager::DefaultLogger(), or separate Logger instances each with different log ids
32  via LoggingManager::CreateLogger.
33 
34  The log id is passed to the ISink instance with the sink determining how the log id is used
35  in the output.
36 
37  LoggingManager
38  * creates the Logger instances used by the application
39  * provides a static default logger instance
40  * owns the log sink instance
41  * applies checks on severity and output of user data
42 
43  The log macros create a Capture instance to capture the information to log.
44  If the severity and/or user filtering settings would prevent logging, no evaluation
45  of the log arguments will occur, so no performance cost beyond the severity and user
46  filtering check.
47 
48  A sink can do further filter as needed.
49 
50 */
